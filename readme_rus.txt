SecChat

Простой и очень защищённый сервер для чата в реальном времени. Не требует никакого специализированного клиентского программного обеспечения, работает с любыми современными броузерами. Позволяет Администратору ресурса приглашать пользователей, создавать для них каналы, в которых могут в реальном времени переписываться 2 и более пользователей.

Почему именно этот чат?

Vkontakte, facebook, одноклассники - неплохая платфрма для чата, но различные правительственные структуры часто закрывают к ним доступ, а также давно могут получить всю информацию о хозяине любой из анкет.

Клиенты многновенных сообщений ICQ и AOL и т.д. - обладают выделенными серверами, доступ к которым вполне можно прекратить на уровне провайдеров.

Системы на основе протокола jabber сложно настраивать и усталавливать - как клиенты, так и серверную часть .

Secchat -просто устанавливается, потребляет мало ресурсов сервера и клиента, не требует установки клиентсокго программного обеспечения - (в качестве клиента подойдёт почти любой современный броузер - Google Crome, Mozilla Firefox, Opera и т.д.), по умолчанию ведёт передачу всех сообщений через HTTPS (защищённый протокол, который достаточно сложно перехватить сниферами). работает на OpenSource программном обеспечении, свободном от "прослушки" и транслирует сообщеняи в реальном времени.

Установка

Загрузите актуальную версию данного пакета
Установите сервер Apache c расширениями PHP+ssl. В сборке XAMPP уже всё настроено и заранее установлено. Дальнейшая инструкция предоставленна именно для этй сборки
В файле /opt/lampp/etc/httpd.conf измените строку DocumentRoot /opt/lampp/htdocs на DocumentRoot /opt/lampp/htdocs/www
В файле /opt/lampp/etc/extra/httpd-ssl.conf также измените строку DocumentRoot /opt/lampp/htdocs на DocumentRoot /opt/lampp/htdocs/www - этим вы включите работу программы через защищённое соединение https
Создайте пустую базу данных для программы - подробнее - см руководство к lampp и mysql
В файле /opt/lampp/htdocs/secchat-classes/secchat.ini укажите реквизиты для доступа к базе данных MySQL - логин, пароль, название базы данных. Если база данных находится на другом сервере, то следует указать is_persistent=1 - так соединение будет оптимизированно для работы с внешней базой данных. Реквизиты указываются в блоке [Settings for connection to databasae]
Рекомендуем также указать, чтобы программа всегда требовала защищённого соединения - напишие force_https=1 в блоке [General security settings]
Сайт с чатом теперь должен заработать - осталось только инициализировать базу данных.
Перейдите по этой локальной ссылке на сайте - https://127.0.0.1/install и инициализируйте базу данных, введя пароль Администратра Сайта (root)
Зайдите на сайт от имени администратора, создайте каналы и пригласите пользователей, сообщите пользователям логины и пароли, и они уже могут общаться на сайте
Настройте права доступа пользователям.
